# Note Template

Use this template for new notes.

```md
# Topic Name

## What It Is

- one-paragraph definition
- what problem it solves

## Why It Matters

- where it shows up in interviews
- what system or modeling pain point it addresses

## Core Idea

- the shortest high-signal explanation

## How It Works

- step-by-step mechanism
- include formulas or pseudocode if they help

## Key Tradeoffs

- strengths
- weaknesses
- when it works well
- when it breaks down

## Common Confusions

- concept A vs concept B
- what people often mix up

## Interview Framing

- a short answer version
- a deeper answer version

## Related Notes

- link nearby notes in this repository
```

## Additional Guidance

- Prefer high information density.
- Keep examples concrete.
- If using formulas, explain them in plain language.
- If covering systems topics, include lifecycle, memory, latency, and throughput angles when relevant.
